# 云开发环境配置指南

## 问题说明

当前错误：`errCode: -501000 | errMsg: [100003] env not exists`

这个错误表示云开发环境不存在或未正确配置。

## 解决方案

### 方案一：使用动态环境选择（推荐）✨

小程序代码已更新为优先使用动态环境选择。这意味着小程序会自动使用微信开发者工具中**当前选择的环境**，无需手动配置环境 ID。

**步骤：**

1. **打开微信开发者工具**
   - 确保已登录微信开发者账号

2. **打开云开发控制台**
   - 点击工具栏上的 **"云开发"** 按钮
   - 或者在菜单栏选择 **"工具"** → **"云开发"**

3. **创建或选择环境**
   - 如果已有环境，在环境列表中选择一个环境
   - 如果没有环境，点击 **"创建环境"** 创建新环境
   - 环境名称可以自定义（如：`xiaoshuo`）

4. **开通 AI 能力**
   - 在云开发控制台中，找到 **"AI"** 或 **"大模型"** 功能
   - 点击开通 AI 能力
   - 选择使用的模型（如：deepseek-r1-0528）
   - 确认开通

5. **重新编译小程序**
   - 保存所有文件
   - 点击 **"编译"** 按钮重新编译小程序
   - 小程序会自动使用当前选择的环境

### 方案二：手动指定环境 ID

如果动态环境不可用，可以手动指定环境 ID：

1. **获取环境 ID**
   - 在云开发控制台中，找到 **"环境 ID"**
   - 复制环境 ID（格式类似：`xxxx-xxxx-xxxx`）

2. **更新代码**
   - 打开 `miniprogram/app.js`
   - 找到 `const envId = 'xiaoshuo-8g2t52vw6be64c9a'`
   - 替换为你的实际环境 ID
   - 保存文件

3. **重新编译**
   - 点击 **"编译"** 按钮重新编译小程序

## 验证配置

配置完成后，重新编译小程序，查看控制台日志：

## 常见问题

### Q: 什么是动态环境选择？

A: 动态环境选择是指小程序自动使用微信开发者工具中当前选择的环境，无需在代码中硬编码环境 ID。这样更方便，因为你可以直接在开发者工具中切换环境。

### Q: 如何知道当前使用的是哪个环境？

A: 在微信开发者工具的云开发控制台中，当前选中的环境就是小程序使用的环境。

### Q: 动态环境选择失败怎么办？

A: 如果动态环境选择失败，代码会自动尝试使用指定的环境 ID。如果仍然失败，请：
1. 检查环境是否存在于云开发控制台中
2. 检查环境是否已开通 AI 能力
3. 检查环境 ID 是否正确

### Q: 如何开通 AI 能力？

A:
1. 在云开发控制台中，找到 **"AI"** 或 **"大模型"** 功能
2. 点击开通
3. 选择使用的模型（如：deepseek-r1-0528）
4. 确认开通

### Q: 环境不存在怎么办？

A:
1. 在微信开发者工具中，点击 **"云开发"** 按钮
2. 点击 **"创建环境"** 创建新环境
3. 开通 AI 能力
4. 确保环境在环境列表中处于选中状态（小程序会自动使用选中的环境）
