# 云函数部署说明（API Key 使用环境变量）

## 重要说明

✅ **API Key 不应写入代码，请使用云函数环境变量配置。**

API Key 示例：`your-api-key-here`。请替换为你自己的密钥，并配置到 `DASHSCOPE_API_KEY` 或 `ALIYUN_API_KEY`。

## 部署步骤

### 第一步：部署云函数

1. **在微信开发者工具中**
   - 找到左侧文件树中的 `cloudfunctions/imageAPI` 文件夹
   - **右键点击** `imageAPI` 文件夹
   - 选择 **上传并部署：云端安装依赖**
   - 等待部署完成（会显示进度条）

2. **验证部署**
   - 点击工具栏的 **云开发** 按钮
   - 进入 **云函数** 页面
   - 确认 `imageAPI` 出现在列表中，状态为"正常"

### 第二步：测试功能

1. **重新编译小程序**
   - 点击工具栏的 **编译** 按钮

2. **测试插画生成**
   - 进入 **插画生成** 页面
   - 输入描述，例如："古风仙侠，白衣剑客"
   - 点击 **生成插画**
   - 等待生成完成

3. **查看日志**
   - 在云开发控制台 → 云函数 → imageAPI → 日志
   - 查看调用记录，应该看到：
     ```
     🔑 使用云函数环境变量中的API Key
     📞 调用阿里云通义万相2.5 API...
     ✅ 阿里云通义万相2.5调用成功
     ```

## 注意事项

⚠️ **安全提示：**
- 不要将真实 API Key 写入代码或提交到 Git
- 请使用云函数环境变量保存 API Key

✅ **当前配置：**
- API Key 从环境变量读取
- 需要配置 `DASHSCOPE_API_KEY` 或 `ALIYUN_API_KEY`
- 部署云函数后即可使用

## 如果遇到问题

### 问题1：云函数部署失败

**解决方法：**
1. 检查网络连接
2. 确认云开发环境已开通
3. 查看部署日志中的错误信息

### 问题2：API调用失败

**查看日志：**
1. 在云开发控制台 → 云函数 → imageAPI → 日志
2. 查找错误信息：
   - 如果看到 `❌ 认证失败`，说明API Key可能无效
   - 如果看到 `❌ 请求参数错误`，查看具体错误信息
   - 如果看到 `❌ 网络请求失败`，检查网络连接

### 问题3：仍然显示占位图片

**检查清单：**
- [ ] 云函数是否已部署
- [ ] 云函数状态是否为"正常"
- [ ] 是否重新编译了小程序
- [ ] 查看云函数日志，确认API是否被调用

## 验证部署成功

部署后，生成插画时应该看到：

**在云函数日志中：**
```
🚀 imageAPI 云函数被调用
🔑 使用云函数环境变量中的API Key
📞 调用阿里云通义万相2.5 API...
✅ 阿里云通义万相2.5调用成功
✅ 图片生成成功
```

**在小程序中：**
- 显示真实生成的图片（不是占位图片）
- 控制台显示"生成成功"

## 完成

部署完成后，插画生成功能即可正常使用！
