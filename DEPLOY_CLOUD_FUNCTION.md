# 云函数部署指南

## 错误说明

如果看到以下错误：
```
errCode: -501000 | errMsg: FunctionName parameter could not be found
```

说明云函数 `imageAPI` 还没有部署到云端。

## 部署步骤

### 方法一：使用微信开发者工具（推荐）

1. **打开微信开发者工具**
   - 确保已登录并打开项目

2. **部署云函数**
   - 在左侧文件树中，找到 `cloudfunctions/imageAPI` 文件夹
   - **右键点击** `imageAPI` 文件夹
   - 选择 **上传并部署：云端安装依赖**
   - 等待部署完成（会显示进度）

3. **验证部署**
   - 部署完成后，在云开发控制台查看
   - 进入 **云开发** → **云函数**
   - 确认 `imageAPI` 出现在云函数列表中
   - 状态应该显示为"正常"

### 方法二：使用命令行（可选）

```bash
# 进入云函数目录
cd cloudfunctions/imageAPI

# 安装依赖
npm install

# 然后在微信开发者工具中右键文件夹选择"上传并部署"
```

## 部署前检查

### 1. 检查云函数文件结构

确保 `cloudfunctions/imageAPI` 目录下有：
- ✅ `index.js` - 云函数入口文件
- ✅ `package.json` - 依赖配置文件

### 2. 检查 package.json

确保 `package.json` 包含：
```json
{
  "dependencies": {
    "wx-server-sdk": "~2.6.3",
    "axios": "^1.6.0"
  }
}
```

### 3. 检查云开发环境

- 确保已开通云开发
- 确保环境ID正确（在 `app.js` 中配置）

## 部署后配置

### 配置环境变量（必需）

1. **打开云开发控制台**
   - 在微信开发者工具中点击 **云开发** 按钮
   - 进入 **云函数** → **imageAPI**

2. **配置环境变量**
   - 点击 **配置** 标签
   - 找到 **环境变量** 部分
   - 点击 **添加环境变量**
   - 添加以下两个变量：
     ```
     变量名: WENXIN_API_KEY
     变量值: 你的API Key
     
     变量名: WENXIN_SECRET_KEY
     变量值: 你的Secret Key
     ```

3. **保存配置**
   - 点击 **保存**
   - 等待配置生效

## 测试部署

### 1. 检查云函数列表

- 在云开发控制台 → **云函数**
- 确认 `imageAPI` 存在且状态为"正常"

### 2. 测试调用

在小程序中：
1. 进入 **插画生成** 页面
2. 输入描述并生成
3. 查看控制台日志：
   - 如果看到 `📞 正在调用云函数 imageAPI...`，说明调用成功
   - 如果仍然报错，检查云函数名称是否正确

## 常见问题

### Q1: 部署失败，提示"依赖安装失败"

**解决方法：**
1. 检查网络连接
2. 在本地先执行 `npm install`
3. 检查 `package.json` 格式是否正确

### Q2: 部署成功但调用失败

**可能原因：**
1. 环境变量未配置
2. API密钥错误
3. 云函数代码有错误

**解决方法：**
1. 检查环境变量配置
2. 查看云函数日志（云开发控制台 → 云函数 → 日志）
3. 验证API密钥是否正确

### Q3: 如何查看云函数日志？

1. 打开云开发控制台
2. 进入 **云函数** → **imageAPI**
3. 点击 **日志** 标签
4. 查看调用记录和错误信息
