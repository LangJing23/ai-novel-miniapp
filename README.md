# AI创作助手小程序 - 完整实现说明

## 项目概述

本项目是一个基于微信小程序的"一站式创作辅助工具链"，集成了4个AI智能体（人物角色定义、世界观构建、人物对话生成、插画生成），为小说作者、剧本创作者等提供全链路创作辅助。

## 核心功能模块

### 1. 工具类模块 (`miniprogram/utils/`)

#### `api.js` - API服务模块
- **功能**：封装所有AI API调用
- **主要函数**：
  - `generateCharacter(description)` - 生成人物角色定义
  - `generateWorldview(description)` - 生成世界观设定
  - `generateDialog(scene, character, worldview)` - 生成对话（支持关联角色和世界观）
  - `generateIllustration(description, character, worldview)` - 生成插画（支持导入人设/世界观）
- **设计思路**：统一API调用接口，便于后续替换不同的AI服务提供商

#### `storage.js` - 本地存储工具
- **功能**：管理所有本地数据存储
- **存储内容**：
  - 用户信息
  - API配置
  - 角色列表
  - 世界观列表
  - 作品列表
- **设计思路**：使用微信小程序的`wx.setStorageSync`实现本地持久化，后续可升级为云数据库

#### `common.js` - 通用工具函数
- **功能**：提供日期格式化、提示框、图片处理等通用功能
- **设计思路**：抽取公共功能，提高代码复用性

### 2. 核心页面实现

#### 创作工作台 (`pages/create/`)
- **功能**：4个智能体的入口页面
- **设计特点**：
  - 使用网格布局展示4个智能体卡片
  - 每个卡片有独特的渐变色背景，视觉区分明显
  - 提供使用提示，降低用户学习成本

#### 人物角色定义 (`pages/character/`)
- **功能流程**：
  1. 输入页面：用户输入角色描述，提供示例提示
  2. AI生成：调用豆包API生成结构化人设卡
  3. 结果展示：展示基础信息、性格、行为逻辑、口头禅、禁忌等
  4. 操作功能：编辑、重新生成、保存、关联对话、导出图片
- **设计特点**：
  - 输入框有字符计数和示例提示
  - 结果以卡片形式展示，结构清晰
  - 支持微调功能（调用API进行二次生成）

#### 世界观构建 (`pages/world/`)
- **功能流程**：类似角色定义，生成世界观设定集
- **输出内容**：时代背景、地理环境、社会规则、核心冲突、科技/魔法水平
- **联动功能**：可关联到对话和插画生成

#### 人物对话生成 (`pages/dialog/`)
- **核心特性**：智能体联动
  - 可选择已创建的角色（确保对话符合人设）
  - 可选择已创建的世界观（确保对话符合场景）
  - 输入对话场景描述
- **设计思路**：通过下拉选择器实现数据互通，避免重复输入

#### 插画生成 (`pages/image/`)
- **核心特性**：
  - 支持导入角色和世界观，自动提取关键词生成提示词
  - 支持文本描述补充
  - 生成后可保存、下载到相册
- **设计思路**：结合角色和世界观信息，生成更贴合设定的插画

#### 我的作品 (`pages/works/`)
- **功能**：
  - 按类型筛选（全部/角色/世界观/对话/插画）
  - 展示作品列表（标题、时间、类型）
  - 查看、删除作品
- **设计特点**：使用图标区分不同类型，时间倒序排列

#### 我的 (`pages/me/`)
- **功能**：
  - 微信一键登录
  - API配置入口
  - 统计信息（角色数、世界观数、作品数）
  - 帮助中心、关于我们
- **设计特点**：渐变背景的用户卡片，功能菜单清晰

#### 模板库 (`pages/templates/`)
- **功能**：提供预设模板，快速开始创作
- **模板类型**：角色模板、世界观模板、对话场景模板、插画风格模板
- **设计思路**：降低新用户使用门槛

### 3. 云函数实现

#### `doubaoAPI` 云函数
- **功能**：调用豆包API进行文本生成
- **实现说明**：
  - 当前为模拟实现（返回模拟数据）
  - 实际部署时需要：
    1. 在豆包开放平台申请API Key
    2. 配置环境变量或从数据库读取API Key
    3. 使用HTTP库（如axios）调用豆包API
    4. 处理API响应和错误

#### `imageAPI` 云函数
- **功能**：调用插画生成API
- **实现说明**：
  - 当前为模拟实现
  - 支持文心一格、百度AI云等平台
  - 实际部署时需要配置对应的API接口

## 技术架构

### 前端技术栈
- **框架**：微信小程序原生框架（WXML + WXSS + JavaScript）
- **状态管理**：使用`data`和`setData`进行本地状态管理
- **数据存储**：本地存储（`wx.setStorageSync`）
- **样式方案**：使用rpx响应式单位，支持不同屏幕尺寸

### 后端技术栈
- **云开发**：微信云开发
- **云函数**：Node.js环境
- **数据库**：预留云数据库接口（当前使用本地存储）

### 数据流转
1. **用户输入** → 页面收集数据
2. **API调用** → 通过云函数调用外部AI API
3. **结果处理** → 解析API响应，格式化数据
4. **数据存储** → 保存到本地存储
5. **数据联动** → 其他页面读取已保存的数据

## 智能体联动机制

### 数据互通实现
1. **角色 → 对话**：
   - 对话页面加载时读取所有已保存的角色
   - 用户选择角色后，生成对话时传入角色信息
   - API调用时，将角色的人设信息作为上下文传入

2. **世界观 → 对话/插画**：
   - 类似角色联动机制
   - 世界观信息作为场景背景传入

3. **角色 + 世界观 → 插画**：
   - 插画页面可同时选择角色和世界观
   - 自动提取关键词（角色身份、性格、世界观风格等）
   - 组合生成更精准的图片提示词

## 关键设计决策

### 1. 为什么使用本地存储而不是云数据库？
- **MVP阶段**：降低开发复杂度，快速验证功能
- **后续升级**：可以无缝迁移到云数据库，只需修改`storage.js`

### 2. 为什么云函数使用模拟数据？
- **开发阶段**：不依赖外部API，可以快速开发前端功能
- **实际部署**：需要替换为真实API调用

### 3. 为什么采用页面级状态管理？
- **简单性**：小程序规模适中，不需要复杂的状态管理
- **性能**：页面级状态管理性能更好
