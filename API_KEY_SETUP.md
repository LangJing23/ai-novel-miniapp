# API Key 配置指南

## 您的配置信息

- **API Key**: `your-api-key-here`
- **模型**: 通义万相2.5-文生图-Preview (`wanx2.5-t2i-preview`)

## 配置步骤

### 方法1：通过环境变量配置（推荐）

1. **打开云开发控制台**
   - 在微信开发者工具中，点击工具栏的 **云开发** 按钮
   - 进入 **云函数** → **imageAPI**

2. **配置环境变量**
   - 点击 **配置** 标签
   - 找到 **环境变量** 部分
   - 点击 **添加环境变量**
   - 添加以下变量：
     ```
     变量名: DASHSCOPE_API_KEY
     变量值: your-api-key-here
     ```
   - 或者使用：
     ```
     变量名: ALIYUN_API_KEY
     变量值: your-api-key-here
     ```
   - 点击 **保存**

3. **部署云函数**
   - 右键点击 `cloudfunctions/imageAPI` 文件夹
   - 选择 **上传并部署：云端安装依赖**
   - 等待部署完成

### 方法2：代码中已设置默认值（临时方案）

代码中已经设置了您的API Key作为默认值，但**强烈建议使用环境变量**，因为：
- 更安全（不会暴露在代码中）
- 更灵活（可以随时更换）
- 更符合最佳实践

## 验证配置

### 1. 检查环境变量

在云开发控制台 → 云函数 → imageAPI → 配置中，确认环境变量已添加。

1. 在小程序中进入 **插画生成** 页面
2. 输入描述，例如："古风仙侠，白衣剑客"
3. 点击 **生成插画**
4. 查看控制台日志：

   - 如果看到 `✅ 阿里云通义万相调用成功`
   - 说明配置成功

### 3. 查看云函数日志

在云开发控制台 → 云函数 → imageAPI → 日志中，查看调用记录：
- 成功时会看到：`✅ 图片生成成功`
- 失败时会看到错误信息

## 常见问题

### Q1: 为什么代码中已经有API Key了，还要配置环境变量？

**A:** 代码中的API Key是作为**默认值**，用于快速测试。但生产环境应该：
- 使用环境变量存储敏感信息
- 避免在代码中硬编码密钥
- 方便更换密钥而不需要修改代码

### Q2: 环境变量配置后需要重新部署吗？

**A:** 是的，配置环境变量后需要：
1. 保存配置
2. 重新部署云函数（右键 → 上传并部署）

### Q3: 如何确认API Key是否有效？

**A:** 生成插画时：
- 如果成功生成图片，说明API Key有效
- 如果返回错误（如401、403），说明API Key无效或过期
- 查看云函数日志获取详细错误信息

### Q4: API Key安全吗？

**A:** 
- ✅ 环境变量存储在云端，相对安全
- ⚠️ 不要在代码中提交API Key到Git仓库
- ⚠️ 不要在前端代码中使用API Key
- ✅ 定期更换API Key
