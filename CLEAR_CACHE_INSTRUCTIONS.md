# 清除缓存并重新编译说明

## 问题描述

如果看到以下错误：
```
api.js:116 云函数调用失败，使用本地模拟
Error: cloud.callFunction:fail Error: errCode: -501000
```

并且**没有看到**以下日志：
```
🚀 API模块已加载 - 版本 2.3.0
```

说明代码没有被重新编译，微信开发者工具仍在使用缓存的旧代码。

## 解决方案

### 方法一：清除文件缓存（推荐）

1. **在微信开发者工具中**：
   - 点击菜单栏 **"工具"** → **"清除缓存"**
   - 选择 **"清除文件缓存"**
   - 点击 **"确定"**

2. **重新编译**：
   - 点击工具栏的 **"编译"** 按钮
   - 或者按快捷键 **Ctrl + R** (Windows) / **Cmd + R** (Mac)

3. **验证代码已更新**：
   - 查看控制台，应该看到：
     ```
     🚀 API模块已加载 - 版本 2.3.0
     📅 更新时间: [时间戳]
     ✨ 特性: 直接使用官方 AI API，完全绕过云函数
     ```

### 方法二：关闭并重新打开项目

1. **关闭微信开发者工具**
2. **重新打开微信开发者工具**
3. **重新打开项目**
4. **点击编译按钮**

### 方法三：清除所有缓存

1. **在微信开发者工具中**：
   - 点击菜单栏 **"工具"** → **"清除缓存"**
   - 选择 **"清除文件缓存"** 和 **"清除数据缓存"**
   - 点击 **"确定"**

2. **重新编译项目**

### 方法四：删除 .wxapkg 文件（如果存在）

1. **关闭微信开发者工具**
2. **删除项目目录下的 `.wxapkg` 文件**（如果存在）
3. **重新打开项目**
4. **重新编译**

## 如果仍然看到云函数错误

如果清除缓存后仍然看到云函数调用错误，请检查：

1. **文件是否已保存**：
   - 确保所有修改的文件都已保存（Ctrl + S）

2. **代码是否正确**：
   - 检查 `miniprogram/utils/api.js` 文件中是否还有 `wx.cloud.callFunction` 调用
   - 使用搜索功能查找 `callFunction`，应该找不到任何结果

3. **版本标记是否存在**：
   - 检查 `miniprogram/utils/api.js` 文件开头是否有版本标记日志
   - 应该看到 `🚀 API模块已加载 - 版本 2.3.0`

4. **重新启动开发者工具**：
   - 完全关闭微信开发者工具
   - 重新打开并加载项目

## 常见问题

### Q: 为什么代码修改后没有生效？

A: 微信开发者工具可能缓存了旧代码。需要清除缓存并重新编译。

### Q: 清除缓存会影响我的数据吗？

A: 清除文件缓存不会影响你的代码和数据，只是清除编译缓存。

### Q: 如何确认代码已更新？

A: 查看控制台日志，如果看到版本 2.3.0 的标记，说明新代码已加载。

### Q: 仍然看到云函数错误怎么办？

A: 
1. 检查代码中是否还有 `wx.cloud.callFunction` 调用
2. 确认所有文件都已保存
3. 完全关闭并重新打开开发者工具
4. 删除 `.wxapkg` 文件（如果存在）
