# 图片生成API配置指南

## 问题说明

如果生成的图片始终是同一个占位图片，说明图片生成功能还在使用本地模拟模式，没有调用真实的API。

## 解决方案

### 第一步：配置文心一格API密钥

1. **获取API密钥**
   - 访问 [百度智能云控制台](https://console.bce.baidu.com/)
   - 登录后进入 **千帆大模型平台**
   - 在 **应用接入** 中创建应用，获取：
     - `API Key`
     - `Secret Key`

2. **在云开发控制台配置环境变量**
   - 打开微信开发者工具
   - 进入 **云开发** → **云函数** → **imageAPI**
   - 点击 **配置** → **环境变量**
   - 添加以下环境变量：
     ```
     WENXIN_API_KEY = 你的API Key
     WENXIN_SECRET_KEY = 你的Secret Key
     ```

### 第二步：部署云函数

1. **安装依赖**
   - 在 `cloudfunctions/imageAPI` 目录下执行：
     ```bash
     npm install
     ```
   - 或者在微信开发者工具中：
     - 右键 `cloudfunctions/imageAPI` 文件夹
     - 选择 **上传并部署：云端安装依赖**

2. **验证部署**
   - 部署成功后，在云开发控制台查看云函数列表
   - 确认 `imageAPI` 云函数状态为"正常"

### 第三步：测试

1. **在小程序中测试**
   - 进入 **插画生成** 页面
   - 输入描述，例如："古风仙侠，白衣剑客"
   - 点击 **生成插画**
   - 查看控制台日志：
     - 如果看到 `📞 正在调用云函数 imageAPI...`，说明正在调用云函数
     - 如果看到 `✅ 图片生成成功`，说明API调用成功
     - 如果看到 `⚠️ 降级到本地模拟模式`，说明API配置有问题

2. **检查日志**
   - 在云开发控制台 → **云函数** → **日志**
   - 查看 `imageAPI` 的调用日志
   - 如果有错误，根据错误信息调整配置

## 常见问题

### Q1: 仍然显示占位图片？

**可能原因：**
1. 环境变量未配置或配置错误
2. 云函数未部署
3. API密钥无效

**解决方法：**
1. 检查环境变量是否正确配置
2. 重新部署云函数
3. 验证API密钥是否有效

### Q2: 云函数调用失败？

**可能原因：**
1. 云函数未部署
2. 网络问题
3. API密钥错误

**解决方法：**
1. 确认云函数已部署
2. 检查网络连接
3. 验证API密钥

### Q3: 如何查看详细的错误信息？

1. 打开微信开发者工具控制台
2. 查看 `callImageAPI` 相关的日志
3. 在云开发控制台查看云函数日志

## 验证配置是否成功

生成插画时，查看控制台日志：

**成功的情况：**
```
callImageAPI 被调用 - 使用云函数调用真实API
📞 正在调用云函数 imageAPI...
📥 云函数返回结果: {...}
✅ 图片生成成功
```

**失败的情况（会降级到模拟模式）：**
```
callImageAPI 被调用 - 使用云函数调用真实API
📞 正在调用云函数 imageAPI...
❌ 调用云函数失败: ...
⚠️ 降级到本地模拟模式
```

## 注意事项

1. **API费用**：文心一格按调用次数计费，建议查看定价
2. **API限制**：注意API的调用频率限制
3. **图片格式**：返回的是base64格式的图片数据，可以直接在小程序中显示

## 下一步

配置完成后，重新测试插画生成功能。如果还有问题，请查看控制台和云函数日志，根据错误信息进行排查。
